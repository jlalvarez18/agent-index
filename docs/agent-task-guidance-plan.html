Agent Task Guidance Mode

A plan to make agent-index task more useful for autonomous agents by turning retrieval output into a compact, confidence-aware next-action playbook.

Primary Goal Increase autonomous-agent wins by making agents use agent-index earlier and more confidently.
Implementation Shape Add opt-in guidance to the existing task command instead of creating a new entrypoint.
Compatibility Preserve existing compact and JSON output unless --agent-guidance is passed.

1. Add A Guidance Data Model

Add an internal AgentTaskGuidance object generated after the task workflow runs. It should summarize what the agent should do next.

recommendedNextAction confidence openFirst why followUpCommands
Reasoning: ranking and guidance are different jobs. Retrieval finds likely code; guidance tells the agent how to act on those results without forcing another round of broad search.

2. Use Deterministic Confidence Rules

Compute high, medium, or low confidence from existing step results.

  • High: top source result exists, has evidence, and related tests or graph neighbors are present.
  • Medium: top source result exists, but evidence or related-test support is weaker.
  • Low: no useful source result, mostly empty steps, or weak/no evidence.
Reasoning: agents need a trust signal, but it must be explainable. A small deterministic rule set is easier to test, benchmark, and debug than hidden scoring magic.

3. Add Guided Compact Output

When --agent-guidance is used with --format compact, prepend a short guidance block before the existing step details. 

Guidance: open-top-result confidence=high
  open: src/click/globals.py:54
  why: source hit rank 1, resolver-like symbol, related tests found
  next: inspect source before broad rg
Reasoning: the current compact output says where to open, but it does not strongly frame the workflow. This block gives agents permission to stop searching and inspect the top result first.

4. Add JSON Guidance For Integrations

When --agent-guidance is used with --format json, include a top-level guidance field next to the existing plan and steps.

Reasoning: compact output helps humans and agents in terminals. JSON guidance gives benchmarks, harnesses, and future integrations a stable machine-readable contract.

5. Keep The CLI Backward Compatible

Existing agent-index task ... --format compact and --format json behavior should remain unchanged unless the new flag is present.

Reasoning: current benchmark artifacts and compact-output consumers should not churn while the new guidance mode is being proven.

6. Test The Agent-Visible Contract

Add unit and CLI tests that verify both structured guidance decisions and the exact compact text agents see.

  • High-confidence bugfix guidance with source and related tests.
  • Medium-confidence source-only guidance.
  • Low-confidence empty or noisy result guidance.
  • source-to-tests guidance opens tests first.
  • Compact output is unchanged without --agent-guidance.
  • Compact and JSON output include guidance with the flag.
Reasoning: this feature is only useful if the emitted instructions are clear and stable. Testing the text is not cosmetic here; it is part of the product behavior.

7. Document And Dogfood

Update the agent query cookbook with the new flag and add one dogfood note showing how the guidance block changes the agent's next move.

Reasoning: adoption is the whole point. Documentation and dogfood notes make the intended behavior visible before running a larger autonomous comparison.

Acceptance Criteria

  • agent-index task ... --format compact remains unchanged by default.
  • agent-index task ... --format compact --agent-guidance prints a concise guidance block.
  • agent-index task ... --format json --agent-guidance emits a structured guidance object.
  • Guidance decisions are deterministic and covered by tests.
  • The feature gives agents a clear next action without forcing blind trust in weak results.
