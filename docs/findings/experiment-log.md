# Agent Index Experiment Log

This log preserves the test-and-benchmark progression for the Graphify v8 prototype. It is intentionally closer to a lab notebook than a polished narrative.

## How To Read This

- `Regression test` names the focused automated test or benchmark check added to catch the behavior.
- `Before` and `After` use the Graphify source-only benchmark when available.
- Split metrics use symbol-first scoring unless the row explicitly says blended.
- A `-` means the metric was not yet available or was not the point of that step.

## Current Conclusion

The expanded experiment supports a narrower claim than "symbol search wins": FTS is still the recall backbone, while symbol structure, query intent, and small coding-domain priors improve ranking and navigability. The product-facing claim is now sharper: an LLM agent can supply structured terms and constraints, and `agent-index` can turn those into ranked code objects more usefully than raw rg-style file search. In the current worktree, the rerun preservation matrix is green for Graphify, HTTPX, Click, Rich, Pytest, FastAPI, Pydantic, Poetry, NetworkX friendly/adversarial, SQLAlchemy friendly/adversarial, Scikit-learn friendly/adversarial, Django friendly/adversarial, Celery friendly/adversarial, Black, Jinja, attrs, h11, wsproto, Trio, and urllib3. This is good agent-navigation evidence, but not a general proof against Graphify: Graphify's token-reduction benchmark measures context compression, while `agent-index` measures ranked exact navigation. On attrs, Black, Jinja, wsproto, and urllib3 same-corpus comparisons, `agent-index` now beats Graphify-style mention rates for exact symbol/file navigation. On Graphify, HTTPX, Click, Rich, Pytest, FastAPI, Pydantic, Poetry, NetworkX, SQLAlchemy, Scikit-learn, Django, Django adversarial, Celery, Celery adversarial, Black, Jinja, attrs, h11, wsproto, Trio, and urllib3 structured-agent runs, `agent-index` beats the rg-style file baseline on top-one file retrieval while also returning exact symbols.
